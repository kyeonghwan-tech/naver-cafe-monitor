## Git이 뭔가요?

**Git**은 파일의 변경 이력을 기록하는 도구입니다.
**GitHub**는 그 이력을 인터넷에 저장하는 클라우드 서비스입니다.

쉽게 말하면:

```
Git    = 문서의 "변경 이력 추적" 기능 (마치 Word의 '변경 내용 추적')
GitHub = 그 이력을 온라인에 백업하고 팀과 공유하는 서비스
커밋   = 지금 상태를 저장하는 행위 ("저장" 버튼 누르기)
푸시   = 저장한 내용을 GitHub에 올리는 행위 ("업로드")
```

## Claude Code에서 Git 사용하기

Claude Code 터미널에서 Git 관련 작업은 모두 프롬프트로 처리합니다.

## 1. 처음 시작 — 프로젝트를 GitHub에 등록하기

프로젝트를 처음 GitHub에 올릴 때 사용합니다.

```
프롬프트:
"지금까지 진행했던 것들을 프로젝트를 만들고 그걸 깃헙에 등록해줘."
```

Claude가 자동으로:
1. GitHub에 새 레포지토리(저장소) 생성
2. 현재 폴더를 Git 저장소로 초기화
3. 모든 파일을 첫 번째 커밋으로 저장
4. GitHub에 업로드(push)

완료 후 GitHub 주소를 알려줍니다. 예: `https://github.com/내아이디/프로젝트명`

---

## 2. 저장하고 올리기 — 커밋 & 푸시

작업 내용을 저장하고 GitHub에 올리는 가장 자주 쓰는 명령입니다.

### 기본 표현

```
프롬프트:
"작업내용 모두 커밋하고 푸시해줘"
```

```
프롬프트:
"지금까지 변경한 내용 저장하고 깃헙에 올려줘"
```

### 특정 파일만 저장할 때

```
프롬프트:
"README.md 파일만 커밋하고 푸시해줘"
```

```
프롬프트:
"아키텍처 문서랑 리드미만 깃헙에 올려줘"
```

### 커밋 메시지를 직접 지정할 때

```
프롬프트:
"'대시보드 UI 추가' 라는 메시지로 커밋하고 푸시해줘"
```

> 💡 **팁:** 커밋 메시지를 직접 지정하지 않아도 됩니다.
> Claude가 변경 내용을 분석해서 적절한 메시지를 자동으로 작성합니다.

## 3. 현재 상태 확인하기

어떤 파일이 변경됐는지, 저장이 필요한지 확인할 때 씁니다.

```
프롬프트:
"지금 변경된 파일이 뭐가 있어?"
```

```
프롬프트:
"깃 상태 확인해줘"
```

```
프롬프트:
"커밋 안 된 파일이 있어?"
```

Claude가 변경된 파일 목록을 보여주고, 커밋이 필요한지 알려줍니다.

## 6. 특정 파일을 Git에서 제외하기

비밀번호, 토큰 등 올리면 안 되는 파일을 제외할 때 씁니다.

```
프롬프트:
".env 파일은 깃헙에 올라가지 않게 해줘"
```

```
프롬프트:
"log 파일들이랑 __pycache__ 폴더는 깃에서 제외해줘"
```

Claude가 `.gitignore` 파일을 자동으로 생성하거나 수정합니다.

> ⚠️ **중요:** `.env` 파일에는 비밀번호, 토큰 등 민감한 정보가 있습니다.
> 반드시 `.gitignore`에 추가해서 GitHub에 올라가지 않도록 해야 합니다.

```

```
프롬프트:
"마지막 커밋 전 상태로 되돌려줘"
```

```

"어제 작업한 상태로 되돌아가고 싶어"

> ⚠️ **주의:** 되돌리기는 신중하게 사용하세요.

## 8. 브랜치 (선택 사항 — 팀 작업 시)

브랜치는 "실험 공간"입니다. 메인 코드를 건드리지 않고 새 기능을 시험해볼 때 씁니다.

```
프롬프트:
"'기능추가-검색' 이라는 브랜치 만들어줘"
```

```
프롬프트:
"현재 브랜치 목록 보여줘"
```

```
프롬프트:
"main 브랜치로 전환해줘"
```

```
프롬프트:
"이 브랜치 작업을 main에 합쳐줘"
```

해야 할 작업이나 버그를 GitHub에 기록할 때 씁니다.

```

"이 프로젝트를 개발자들이 개발할 수 있도록 계획문서를 작성하고
 깃헙 이슈에 등록해줘.
 깃헙 이슈에는 작업 배경, 작업 내용, 인수 조건, 다른 작업과의 의존성이 포함되어야 해"
```

```

"'모바일 UI 지원' 기능을 깃헙 이슈로 등록해줘"
```

```
프롬프트:
"현재 깃헙 이슈 목록 보여줘"
```

## Git 관련 용어 사전

| 용어 | 쉬운 설명 |
|------|----------|
| **레포지토리 (Repo)** | 프로젝트 폴더 (GitHub 상의 저장 공간) |
| **커밋 (Commit)** | 현재 상태를 저장하는 것 ("저장 버튼") |
| **푸시 (Push)** | 저장한 내용을 GitHub에 올리는 것 ("업로드") |
| **풀 (Pull)** | GitHub에서 최신 내용을 받아오는 것 ("다운로드") |
| **브랜치 (Branch)** | 실험용 작업 공간 (원본은 그대로 두고 복사본에서 작업) |
| **머지 (Merge)** | 브랜치의 작업을 메인에 합치는 것 |
| **클론 (Clone)** | GitHub에 있는 프로젝트를 내 컴퓨터로 복사하는 것 |
| **.gitignore** | "이 파일들은 Git에서 무시해줘" 목록 파일 |
| **HEAD** | 현재 내가 보고 있는 버전 |
| **main / master** | 최종 완성본이 있는 기본 브랜치 |

## 실수했을 때 대처법

### "실수로 잘못된 내용을 커밋했어요"

```
프롬프트:
"마지막 커밋을 취소하고 싶어. 파일은 그대로 두고 커밋만 없애줘"
```

### "실수로 .env 같은 중요 파일을 올렸어요"

```
프롬프트:
"깃헙에 실수로 올라간 .env 파일을 이력에서 완전히 제거해줘"
```

> ⚠️ 민감한 정보가 올라간 경우, 파일을 제거해도 이미 본 사람이 있을 수 있습니다.
> 반드시 토큰이나 비밀번호를 새것으로 재발급받으세요.

### "뭔가 꼬인 것 같아요, 처음부터 다시 시작하고 싶어요"

```
프롬프트:
"깃 상태가 이상한 것 같아. 현재 상황을 진단하고 해결해줘"
```

Claude가 현재 상태를 확인하고 안전한 방법으로 해결해줍니다.

## 꼭 기억할 3가지

```
1️⃣ .env 파일은 절대 GitHub에 올리지 않는다
   → ".env 파일 깃에서 제외해줘" 프롬프트 사용

2️⃣ 작업이 끝나면 커밋 & 푸시 습관 들이기
   → "작업내용 모두 커밋하고 푸시해줘" 프롬프트 사용

3️⃣ 큰 변경 전에는 현재 상태 먼저 커밋
   → "일단 지금 상태 저장해줘" 프롬프트 사용 (나중에 돌아올 수 있게)
```
