# 비개발자를 위한 바이브 코딩 튜토리얼
## — 실제 프로젝트로 배우는 AI와 함께하는 앱 개발 —

> 이 튜토리얼은 실제로 만들어진 **"네이버 카페 IT 교육 게시글 모니터"** 프로젝트를 사례로,
> 코딩을 전혀 몰라도 Claude Code와 함께 실용적인 업무 자동화 앱을 만드는 방법을 안내합니다.

---

## 바이브 코딩이란?

**바이브 코딩(Vibe Coding)**이란 코드를 직접 쓰는 대신,
**AI에게 자연어로 원하는 것을 설명하고** AI가 코드를 만들어주는 방식입니다.

```
기존 방식:  아이디어 → 몇 달 공부 → 코딩 → 앱 완성
바이브 코딩: 아이디어 → Claude에게 설명 → 앱 완성 (몇 시간~며칠)
```

비개발자도 할 수 있는 이유는 단 하나 —
**"무엇을 만들고 싶은지"를 잘 설명하면 된다**는 것입니다.

---

## 이 튜토리얼에서 만든 것

```
📋 네이버 카페 IT 교육 게시글 모니터

[네이버 카페 ak573]
    → 10분마다 자동 순회
    → IT 교육 키워드가 포함된 게시글 탐지
    → GitHub에 자동 저장 (기록용)
    → 웹 대시보드에서 영업팀이 응대 현황 관리
```

이 앱이 만들어진 전체 과정을 단계별로 따라가 보겠습니다.

---

## STEP 0. 환경 준비

### Claude Code 설치

Claude Code는 터미널(명령 프롬프트)에서 실행하는 AI 코딩 도우미입니다.

```
프롬프트 예시:
"클로드 코드 설치해줘"
```

Claude Code가 설치되면 터미널에서 `claude` 명령어로 실행할 수 있습니다.

---

### YOLO 모드 설정 (선택 사항)

매번 "이 파일을 만들어도 될까요?"라고 물어보는 확인창이 번거롭다면,
자동 승인 모드를 켤 수 있습니다.

```
프롬프트 예시:
"Claude Code CLI를 YOLO 모드로 실행되게 설정해줘"
```

> ⚠️ YOLO 모드에서는 Claude가 모든 파일 조작을 자동으로 실행합니다.
> 처음엔 기본 모드로 사용하다가 익숙해지면 설정하세요.

## STEP 1. 아이디어를 구체적인 요구사항으로 만들기 (PRD 인터뷰)

가장 중요한 단계입니다. Claude에게 단순히 "앱 만들어줘"라고 하면
Claude가 추측으로 만들어 내가 원하는 것과 달라질 수 있습니다.

**PRD(Product Requirements Document)** — 제품 요구사항 문서를 먼저 작성하면
Claude가 정확하게 만들 수 있습니다.

### 인터뷰 방식으로 PRD 만들기

```
프롬프트 예시:
"구체적으로 이 프로젝트의 PRD를 만들기 위한 심층 인터뷰를 진행해줘."
```

Claude가 질문을 하나씩 던집니다. 아래는 실제 인터뷰 예시입니다.

---

**Claude:** 현재 이 업무를 어떻게 처리하고 계신가요?

> **나:** 카페를 직접 들어가서 게시글을 확인하고 있어요. 반복 업무가 너무 많아서요.

**Claude:** 발견했을 때 어떤 조치를 취하시나요?

> **나:** 댓글을 달거나 이메일로 영업 제안을 보내요.

**Claude:** 하루에 몇 건이나 발견하시나요?

> **나:** 많으면 2~3건, 적으면 없는 날도 있어요.

**Claude:** 탐지 주기는 어떻게 되길 원하세요?

> **나:** 10분 간격이면 좋겠어요.

**Claude:** 알림은 어떻게 받고 싶으신가요?

> **나:** 대시보드가 있으면 이메일 알림은 필요 없을 것 같아요.

---

이런 식으로 10~15개 질문에 답하면 PRD가 완성됩니다.

> 💡 **팁:** 모르거나 아직 생각 안 한 것은 "모르겠어요" 또는 "그건 나중에 정해요"라고
> 자유롭게 답해도 됩니다. Claude가 합리적인 기본값을 제안해줍니다.

---

## STEP 2. 목업(화면 시안) 만들기

PRD가 완성되면 실제 구현 전에 **화면이 어떻게 생겼을지** 먼저 확인합니다.
마치 집을 짓기 전에 설계도면을 보는 것과 같습니다.

```
프롬프트 예시:
"완성된 PRD에 기반해서 목업을 Claude Preview를 사용해서 만들어줘."
```

브라우저에 미리보기 화면이 뜹니다. 여기서 마음에 들지 않는 부분을 말로 수정할 수 있습니다.

```
프롬프트 예시 (수정 요청):
"게시 시간을 '3시간 전', '어제' 같은 상대적 시간으로 표시해줘"
"원문 바로 열기 버튼을 추가해줘"
"당신이 봤을 때 더 좋은 방법으로 수정할 것이 있나요?"
```

> 💡 **팁:** "더 좋은 방법이 있어?" 또는 "개선할 점이 있어?"라고 물으면
> Claude가 UX 관점에서 먼저 제안해 줍니다.

## STEP 3. 외부 서비스 연결 설정

앱이 네이버, GitHub 같은 외부 서비스와 연결되려면 **인증 정보**가 필요합니다.

### 네이버 로그인 쿠키 추출

Claude가 직접 네이버에 로그인할 수는 없으므로,
브라우저에서 로그인 쿠키를 직접 추출해서 전달합니다.

**Chrome에서 쿠키 추출 방법:**
1. 네이버에 로그인
2. F12 (개발자 도구 열기)
3. Application → Cookies → https://naver.com
4. `NID_AUT` 값 복사 → Claude에게 전달
5. `NID_SES` 값 복사 → Claude에게 전달

```
프롬프트 예시:
"NID_AUT 값은 [복사한 값]이고 NID_SES 값은 [복사한 값]이야. .env 파일에 저장해줘."
```

### GitHub 토큰 발급

GitHub는 앱이 게시글을 자동으로 저장할 저장소입니다.
앱이 GitHub에 접근하려면 토큰(비밀번호 같은 것)이 필요합니다.

1. GitHub 로그인 → Settings → Developer settings
2. Personal access tokens → Generate new token
3. `repo` 권한 체크 → 토큰 복사

```
프롬프트 예시:
"GitHub 토큰은 [복사한 토큰]이야. 레포지토리 이름은 naver-cafe-monitor로 해줘."
```

> ⚠️ **보안 주의:** 토큰과 쿠키는 비밀번호와 같습니다. 다른 사람에게 공유하지 마세요.

---

## STEP 4. 실제 앱 구현

목업이 확정되고 외부 서비스 연결이 완료되면 실제 앱을 만듭니다.

```
프롬프트 예시:
"네 구현해주세요"
```

Claude가 자동으로:
- 파일 생성 (`scraper.py`, `db.py`, `api.py`, `main.py` 등)
- 데이터베이스 설계
- API 서버 구축
- 웹 대시보드 UI 제작

을 순서대로 진행합니다.

> 💡 **팁:** 중간에 오류가 나도 당황하지 마세요.
> 오류 메시지를 그대로 Claude에게 붙여넣으면 자동으로 수정해줍니다.

---

## STEP 5. 오류 해결 (실전 사례)

바이브 코딩에서 오류는 당연히 발생합니다. 중요한 건 **해결 방법**입니다.

### 실제 발생한 오류들

**① 인코딩 오류**
```
오류 메시지: UnicodeEncodeError: 'cp949' codec can't encode character
```
```
해결 프롬프트: "이런 오류가 났어: [오류 메시지 붙여넣기]"
```

**② 토큰 인증 실패**
```
오류 메시지: 401 Unauthorized
```
```
해결 프롬프트: "GitHub 토큰이 401 오류가 나. 새 토큰으로 바꿨어: [새 토큰]"
```

**③ 권한 오류**
```
오류 메시지: Access is denied
```
```
해결 프롬프트: "관리자 권한 없이도 되는 다른 방법으로 해줘"
```

> 💡 **핵심 원칙:** 오류가 나면 **오류 메시지 전체를 복사**해서 Claude에게 그대로 주세요.
> "이거 왜 안 돼?"보다 오류 메시지를 주는 게 100배 빠릅니다.

## STEP 6. 기능 추가 및 변경

앱이 완성된 후에도 필요에 따라 기능을 추가하거나 변경할 수 있습니다.

### 실제 변경 사례

```
프롬프트 예시들:

"탐지 주기를 30분에서 10분으로 바꿔줘"

"PC 켤 때마다 자동 시작하게 해줘"

"이메일 알림은 필요 없을 것 같아. 대시보드로 대신할게"

```

기능 추가가 마치 "이거 해줘"라고 말하는 것처럼 간단합니다.

## 바이브 코딩 핵심 원칙

### ✅ 잘 되는 프롬프트

| 상황 | 좋은 프롬프트 예시 |
|------|------------------|
| 기능 요청 | "10분마다 네이버 카페를 자동으로 확인하고 싶어" |
| 화면 수정 | "날짜를 '3시간 전'처럼 상대 시간으로 보여줘" |
| 오류 해결 | "이런 오류가 났어: [오류 메시지 전체 붙여넣기]" |
| 개선 요청 | "더 좋은 방법이 있으면 제안해줘" |
| 조건 변경 | "이메일 대신 웹 대시보드로 바꿔줘" |

### ❌ 잘 안 되는 프롬프트

| 상황 | 나쁜 프롬프트 예시 | 이유 |
|------|-----------------|------|
| 너무 막연 | "좋은 앱 만들어줘" | 뭘 원하는지 모름 |
| 오류 설명 부족 | "안 돼" | 어떻게 안 되는지 모름 |
| 전문용어 강요 | "REST API를 HATEOAS 방식으로 바꿔줘" | 비개발자는 몰라도 됨 — 결과만 설명하면 됨 |

## 바이브 코딩 전체 흐름 요약

```
1. 아이디어 떠오름
        ↓
2. Claude에게 인터뷰 요청 → PRD 완성
        ↓
3. 목업(화면 시안) 확인 → 피드백 반영
        ↓
4. 외부 서비스 연결 (쿠키, 토큰 발급)
        ↓
5. "구현해줘" → Claude가 코드 작성
        ↓
6. 오류 발생 → 오류 메시지 붙여넣기 → 자동 수정
        ↓
7. 추가 기능 요청 → 반복
        ↓
8. 문서화 → GitHub에 저장
        ↓
✅ 완성된 앱 사용
```

## 다음 단계: 내 업무에 적용하기

이 튜토리얼을 보고 "나도 이런 거 만들 수 있겠다"는 생각이 든다면,
내 업무에서 **반복하는 것, 귀찮은 것, 시간 낭비되는 것**을 찾아보세요.

**바이브 코딩으로 자동화하기 좋은 업무 유형:**

- 📋 주기적으로 특정 사이트에서 정보 수집하는 업무
- 📊 엑셀 데이터를 가공해서 보고서 만드는 업무
- 📧 같은 내용의 이메일을 반복해서 보내는 업무
- 🔍 여러 곳에서 데이터를 모아서 비교하는 업무
- 📁 파일 이름 변경, 정리, 분류하는 업무

Claude에게 이렇게 시작해 보세요:

```
"나는 매일 [반복하는 업무]를 하는데, 이걸 자동화하고 싶어.
지금은 [현재 하는 방식]으로 하고 있어.
어떻게 시작하면 될까?"
```
